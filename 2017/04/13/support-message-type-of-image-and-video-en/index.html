Adding Image and Video Sending Function to Bots

mukaiu Follow Apr 13, 2017 ¬∑ 6 mins read

For company activities, we needed to manage users joining groups and provide automatic responses. Earlier at Node Party Beijing, I was exposed to @huan‚Äôs sharing, and Wechaty was perfect for supporting this activity with Docker deployment - it‚Äôs a great Bot Framework.

During development, I discovered that #4 Support Message Type of Image/Video functionality was not yet implemented, so I decided to complete this feature.

1. Problem Analysis

To understand how Web WeChat sends images, I performed data packet capture and analyzed WeChat Web source code:

In Web, Messages are created through this function. Image messages get a MediaId property added, so if we can obtain the MediaId of the file to be uploaded, we can directly call sendMessage to send images.

2. Image Upload

Through packet capture, I found that images are sent via POST request to:

https://file.wx.qq.com/cgi-bin/mmwebwx-bin/webwxuploadmedia?f=json

Searching the source code, I found:

API_webwxuploadmedia: "https://" + o + "/cgi-bin/mmwebwx-bin/webwxuploadmedia",
API_webwxpreview: "/cgi-bin/mmwebwx-bin/webwxpreview",

API_webwxuploadmedia is the image upload address. By searching globally for this variable, could I find the file upload method? After searching:

3. Integrating with Wechaty

To quickly verify feasibility, I directly added Wechaty.sendMedia. After discussing with @huan @lijiarui, we decided to use the form say(MediaMessage(filename)) to send media files. Overloaded:

4. Pitfalls

  1. During testing, I found that sending images sometimes failed because MediaId couldn‚Äôt be obtained. My first thought was: are there still details I haven‚Äôt discovered? Comparing post data, everything was identical with no issues, so where was the problem?

    Later, looking at the source code, I discovered:

    Turns out there‚Äôs another address wx2.qq.com. The corresponding file upload address is file2.wx.qq.com. Not careful enough!

  2. Another pitfall is that WeChat Web has a 20MB limit for videos, which I didn‚Äôt notice initially. Sending large videos will fail.
  3. Circular dependency Since MediaMessage inherits from Message, and Message.say(MediaMessage) needs to reference MediaMessage. OMG, circular reference! TypeScript reported errors and doesn‚Äôt support this approach~ So I moved MediaMessage into message.ts, deleted media-message.ts, and magically created 186 lines of changesüòä
